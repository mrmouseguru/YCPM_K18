# YCPM_K18 - template

## Purpose
This repository stores **User Stories and Acceptance Tests**
as **living specifications**, following  Agile

## Rules
- Acceptance Tests define **what the system does**
- Code exists only to make Acceptance Tests pass
- No feature is considered DONE until Acceptance Tests pass

## Roles
- Business Analyst: writes happy paths
- QA: writes unhappy paths
- Developer: automates and runs tests

## Submission
- User Stories: /requirements/user-stories
- Acceptance Tests: /requirements/acceptance-tests
- Source code: /implementation
